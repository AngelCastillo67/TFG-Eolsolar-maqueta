# Caso de prueba TC-02: Detección FIM sobre /etc/hosts en srv-it

## Trazabilidad

- Escenario que mitiga: A.2 — Modificación no autorizada de ficheros críticos del sistema.
- Requisitos que verifica: RF-04, RF-05.
- Control validado: File Integrity Monitoring mediante agente Wazuh.

## Fecha y hora de ejecución

28/06/2026 - HH:MM

## Preparación

Estado inicial del laboratorio:

- VM `srv-it` activa en Z2 con IP `10.10.20.10`.
- VM `wazuh` activa en Z2 con IP `10.10.20.20`.
- Agente Wazuh de `srv-it` activo.
- Configuración centralizada de Wazuh aplicada al grupo `default`.
- Monitorización FIM habilitada sobre `/etc/hosts`.

Configuración esperada en `agent.conf`:

```xml
<agent_config>
  <syscheck>
    <disabled>no</disabled>
    <frequency>300</frequency>
    <scan_on_start>yes</scan_on_start>
    <alert_new_files>yes</alert_new_files>

    <directories realtime="yes" check_all="yes">/etc/hosts</directories>
    <directories realtime="yes" check_all="yes">/etc</directories>
    <directories realtime="yes" check_all="yes">/usr/bin</directories>
    <directories realtime="yes" check_all="yes">/usr/sbin</directories>

    <ignore>/etc/random-seed</ignore>
    <ignore>/etc/random.seed</ignore>
    <ignore>/etc/adjtime</ignore>
    <ignore>/etc/mtab</ignore>

    <skip_nfs>yes</skip_nfs>
    <skip_dev>yes</skip_dev>
    <skip_proc>yes</skip_proc>
    <skip_sys>yes</skip_sys>
  </syscheck>
</agent_config>
```

## Procedimiento

1. En `srv-it`, crear una copia de seguridad de `/etc/hosts`.

```bash
sudo cp /etc/hosts /etc/hosts.bak
```

2. Verificar que la copia se ha creado correctamente.

```bash
ls -l /etc/hosts /etc/hosts.bak
```

3. Modificar `/etc/hosts` añadiendo una línea de prueba.

```bash
echo "# TC-02 prueba FIM Wazuh $(date)" | sudo tee -a /etc/hosts
```

4. Verificar localmente que el fichero fue modificado.

```bash
tail -n 5 /etc/hosts
```

5. Revisar la detección en Wazuh.

Ruta recomendada:

```text
Wazuh → File Integrity Monitoring
```

También puede revisarse en:

```text
Modules → Security events
```

Búsqueda recomendada:

```text
/etc/hosts
```

6. Restaurar el fichero original una vez obtenidas las evidencias.

```bash
sudo cp /etc/hosts.bak /etc/hosts
```

7. Verificar que se ha restaurado correctamente.

```bash
tail -n 5 /etc/hosts
```

## Resultado esperado

- El fichero `/etc/hosts` debe modificarse correctamente en `srv-it`.
- Wazuh debe detectar la modificación mediante FIM.
- La alerta debe asociarse al agente `srv-it`.
- Debe verse referencia al fichero `/etc/hosts` en Wazuh.

## Notas adicionales

La ruta correcta del fichero es `/etc/hosts`, sin barra final. La ruta `/etc/hosts/` es incorrecta porque Linux interpretaría `hosts` como si fuera un directorio.
