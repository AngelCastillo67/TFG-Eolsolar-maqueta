# Caso de prueba TC-01: Bloqueo de flujo Z2 → Z6

## Trazabilidad

- Escenario que mitiga: A.1 — Comunicación no autorizada desde la zona IT/Z2 hacia la zona OT/Z6.
- Requisitos que verifica: RF-01, RF-02, RF-03.
- Control validado: segmentación de red mediante pfSense y generación de alerta en Wazuh.

## Fecha y hora de ejecución

28/06/2026 - HH:MM

## Preparación

Estado inicial del laboratorio:

- VM `pfsense` activa.
- VM `srv-it` activa en Z2 con IP `10.10.20.10`.
- VM `srv-ot` activa en Z6 con IP `10.10.60.10`.
- VM `wazuh` activa en Z2 con IP `10.10.20.20`.
- Wazuh Dashboard accesible desde el equipo anfitrión mediante NAT: `https://172.20.10.4:8443`.
- Regla de pfSense activa: `DENEGACIÓN Z2 → Z6 (TC-01)`.
- Regla local de Wazuh activa:
  - Rule ID: `100201`
  - Nivel: `12`
  - Descripción: `TC-01: bloqueo Z2 hacia Z6 detectado en pfSense`

Regla local usada en Wazuh:

```xml
<group name="local,pfsense,firewall,tc01,">

  <rule id="100200" level="0">
    <match>filterlog</match>
    <description>pfSense firewall log recibido por syslog</description>
  </rule>

  <rule id="100201" level="12">
    <if_sid>100200</if_sid>
    <match>10.10.20.10,10.10.60.10</match>
    <description>TC-01: bloqueo Z2 hacia Z6 detectado en pfSense</description>
  </rule>

</group>
```

## Procedimiento

1. Desde `srv-it`, comprobar conectividad ICMP hacia `srv-ot`.

```bash
ping -c 5 10.10.60.10
```

2. Desde `srv-it`, comprobar conectividad TCP hacia SSH de `srv-ot`.

```bash
nc -zv -w 5 10.10.60.10 22
```

3. Revisar los logs de firewall en pfSense.

Ruta:

```text
Status → System Logs → Firewall
```

Comprobar que aparece la regla:

```text
DENEGACIÓN Z2 → Z6 (TC-01)
```

con tráfico:

```text
10.10.20.10 → 10.10.60.10
```

4. Revisar las alertas en Wazuh.

Ruta:

```text
Modules → Security events
```

Búsqueda recomendada:

```text
rule.id:100201
```

o:

```text
TC-01
```

## Resultado esperado

- El tráfico desde `srv-it` hacia `srv-ot` debe ser bloqueado.
- `ping` debe fallar.
- `nc` debe fallar o finalizar por timeout.
- pfSense debe registrar el bloqueo mediante la regla `DENEGACIÓN Z2 → Z6 (TC-01)`.
- Wazuh debe generar una alerta de nivel 12 con Rule ID `100201`.

## Resultado obtenido

Pendiente de completar tras la ejecución.

Ejemplo:

```text
El tráfico desde srv-it hacia srv-ot fue bloqueado correctamente. pfSense registró el evento de bloqueo y Wazuh generó una alerta específica con Rule ID 100201.
```
